What the system does

It is not a chat client and not a full CRM. It is a filtering, prioritization and alerting layer for Telegram communication.

🎯

Detects important signals

Finds leads, complaints, booking requests and urgent messages without forcing you to read every incoming message manually.

🧠

Uses AI only when useful

Clear messages are handled by fast rule-based logic. AI is used as an additional layer for unclear cases, better summaries, priority scoring and extracted data.

⚙️

Controls AI usage per profile

Each monitoring profile can have its own AI daily call limit. This keeps costs predictable and lets users decide how much AI analysis each profile should use.

How it works

Connect Telegram, configure what matters, set optional AI limits, then work with prioritized events.

1

Connect Telegram

Add a Telegram bot token to a monitoring profile. Credentials are encrypted and not shown again. Alerts can be activated by sending /start_alerts to the bot from the destination chat.

2

Configure signals and limits

Choose what to track: leads, complaints, requests, urgent messages or general activity. Optionally set a profile-level AI daily call limit for tighter cost control.

3

Review prioritized events

The dashboard shows structured events instead of raw message noise. Important and urgent events can trigger Telegram alerts.

Bot access only

Messaging Monitoring works with the Telegram bot you connect. It does not connect to your personal Telegram account.

🤖

Telegram bot, not account access

The system uses the bot token provided by the user. It cannot read your private Telegram chats, contacts or account history.

📥

Only incoming bot messages

The system stores only messages and technical metadata that are received by the connected bot, such as chat ID, message ID, sender label, message text and received time.

🔐

Encrypted bot credentials

Bot credentials are encrypted before storage. Alert destination settings are stored only to know where internal notifications should be sent.

Rules-first, AI-assisted

The system is designed to avoid unnecessary AI calls. This makes the MVP cheaper, faster and easier to reason about.

⚡

Fast rule-based checks

Obvious greetings, short replies, spam-like messages and clear keyword signals can be handled without calling AI.

❓

AI for ambiguous messages

If a message is unclear, has mixed intent or needs structured extraction, AI can classify it and generate a concise summary.

📊

Predictable usage

Global account limits and profile-level limits help prevent uncontrolled AI usage and make the system safer for production.
